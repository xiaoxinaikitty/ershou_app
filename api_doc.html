商品模块

添加发布商品位置

接口描述：添加发布商品时选择的位置信息(省市区)，插入到数据库中，并返回插入记录的ID

请求路径：/product/location/add

请求方法：POST

请求类型：application/json

请求头：

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

请求数据示例：

{
    "province": "广东省",
    "city": "深圳市",
    "district": "南山区"
}

请求数据模型：

// ReleaseLocationDao.java
public class ReleaseLocationDao {
    
    /**
     * 省份名称
     */
    @NotBlank(message = "省份不能为空")
    private String province;
    
    /**
     * 城市名称
     */
    @NotBlank(message = "城市不能为空")
    private String city;
    
    /**
     * 区/县名称
     */
    @NotBlank(message = "区/县不能为空")
    private String district;
}

响应参数说明：

参数名 类型 说明
code int 状态码，0表示成功
message string 提示信息
data object 返回数据
data.id int 位置记录ID
data.province string 省份名称
data.city string 城市名称
data.district string 区/县名称

响应数据模型：

// ReleaseLocationVO.java
public class ReleaseLocationVO {
    
    /**
     * 位置记录ID
     */
    private Integer id;
    
    /**
     * 省份名称
     */
    private String province;
    
    /**
     * 城市名称
     */
    private String city;
    
    /**
     * 区/县名称
     */
    private String district;
}

响应示例：

{
    "code": 0,
    "data": {
        "id": 1,
        "province": "广东省",
        "city": "深圳市",
        "district": "南山区"
    },
    "message": "success"
}

错误码说明：

错误码 说明
0 成功
40000 请求参数错误
40100 未授权
50000 系统内部错误

备注：

  1. 需要用户登录后访问此接口
  2. 此接口适用于发布商品时添加商品所在位置信息
  3. 返回的位置记录ID可用于商品表的location字段

分页查询商品列表

接口描述：分页获取商品列表，支持多种筛选条件和排序方式

请求路径：/product/list

请求方法：GET

请求参数说明：

参数名 必选 类型 说明
pageNum 否 int 当前页码，从1开始，默认1
pageSize 否 int 每页数量，默认10，最大50
keyword 否 string 商品标题关键词搜索
categoryId 否 int 商品分类ID筛选
minPrice 否 double 价格下限筛选
maxPrice 否 double 价格上限筛选
sortField 否 string 排序字段(price-价格, time-时间, view-浏览量)，默认time
sortOrder 否 string 排序方式(asc-升序, desc-降序)，默认desc
status 否 int 商品状态筛选(0下架 1在售 2已售)，默认1
minConditionLevel 否 int 最低成色等级筛选(1-10)
location 否 string 地区筛选
sellerId 否 long 卖家用户ID筛选

请求数据模型：

// ProductPageQueryDao.java
public class ProductPageQueryDao {
    
    /**
     * 当前页码，从1开始
     */
    private Integer pageNum = 1;
    
    /**
     * 每页数量，默认10
     */
    private Integer pageSize = 10;
    
    /**
     * 商品标题关键词
     */
    private String keyword;
    
    /**
     * 商品分类ID
     */
    private Integer categoryId;
    
    /**
     * 价格下限
     */
    private Double minPrice;
    
    /**
     * 价格上限
     */
    private Double maxPrice;
    
    /**
     * 排序字段 (price-价格, time-时间, view-浏览量)
     */
    private String sortField = "time";
    
    /**
     * 排序方式 (asc-升序, desc-降序)
     */
    private String sortOrder = "desc";
    
    /**
     * 商品状态筛选 (0下架 1在售 2已售)
     */
    private Integer status = 1;
    
    /**
     * 最低成色等级(1-10)
     */
    private Integer minConditionLevel;
    
    /**
     * 地区筛选
     */
    private String location;
    
    /**
     * 卖家ID
     */
    private Long sellerId;
}

请求示例：

GET /product/list?pageNum=1&pageSize=10&categoryId=2&minPrice=1000&maxPrice=5000&sortField=price&sortOrder=asc

响应参数说明：

参数名 类型 说明
code int 状态码，0表示成功
message string 提示信息
data object 分页结果数据
data.pageNum int 当前页码
data.pageSize int 每页大小
data.total long 总记录数
data.pages int 总页数
data.list array 当前页商品列表
data.hasPrevious boolean 是否有上一页
data.hasNext boolean 是否有下一页

商品列表项参数说明：

参数名 类型 说明
productId long 商品ID
title string 商品标题
price decimal 商品价格
originalPrice decimal 商品原价
categoryId int 分类ID
categoryName string 分类名称
userId long 发布用户ID
username string 发布用户名
conditionLevel int 物品成色(1-10级)
location string 商品所在地
viewCount int 浏览次数
createdTime datetime 发布时间
mainImageUrl string 商品主图URL

响应数据模型：

备注：

  1. 分页查询默认返回在售(status=1)的商品
  2. 如果不指定排序规则，默认按发布时间降序排列（最新发布的商品排在前面）
  3. 每页最多返回50条数据，超过限制会自动调整为50
  4. categoryId对应的分类名称会自动关联查询并返回
  5. 商品主图(is_main=1)会自动关联查询并返回

冻结/解冻钱包账户余额

接口描述：根据业务需求，对钱包账户的部分余额进行冻结或解冻操作，并更新相应的frozen_balance字段和last_update_time

请求路径：/api/wallet/freeze

请求方法：PUT

请求类型：application/json

请求头：

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

请求数据示例：

{
    "accountId": 123,
    "operationType": 1,
    "amount": 500.00
}

请求数据模型：

// FreezeWalletDTO.java
public class FreezeWalletDTO {
    
    /**
     * 钱包账户ID
     */
    @NotNull(message = "账户ID不能为空")
    private Long accountId;

    /**
     * 操作类型（1-冻结，2-解冻）
     */
    @NotNull(message = "操作类型不能为空")
    private Integer operationType;

    /**
     * 要冻结或解冻的金额
     */
    @NotNull(message = "金额不能为空")
    @Min(value = 0, message = "金额必须大于等于0")
    private BigDecimal amount;
}

响应参数说明：

参数名 类型 说明
code int 状态码，0表示成功
message string 提示信息
data object 更新后的钱包账户信息对象
data.accountId long 钱包账户ID
data.userId long 用户ID
data.balance decimal 更新后的账户余额
data.frozenBalance decimal 更新后的冻结余额
data.lastUpdateTime datetime 最后更新时间

{
    "code": 0,
    "data": {
        "accountId": 123,
        "userId": 1,

    },

{
    "code": 0,
    "data": {
        "accountId": 123,
        "userId": 1,
        "balance": 2000.00,
        "frozenBalance": 0.00,
        "lastUpdateTime": "2023-06-15T16:50:15"
    },
    "message": "success"

错误码说明：

错误码 说明
0 成功
40000 请求参数错误
40100 未授权
40400 钱包账户不存在
50001 操作失败（如余额不足、冻结余额不足等）
50000 系统内部错误

备注：

  1. 此接口用于对钱包账户的余额进行冻结或解冻操作
  2. 操作类型(operationType)：1表示冻结操作，2表示解冻操作
  3. 冻结操作会将指定金额从可用余额(balance)转移到冻结余额(frozenBalance)
  4. 解冻操作会将指定金额从冻结余额(frozenBalance)转移到可用余额(balance)
  5. 操作金额必须大于0，且不能超过相应余额
  6. 每次操作都会自动更新最后更新时间
  7. 需要用户登录并且有权限操作该钱包账户

请求头：

Content-Type: application/json
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

请求参数说明：

参数名 必选 类型 说明
userId 是 long 用户ID
paymentPassword 是 string 要设置的支付密码，长度为6-20位
oldPaymentPassword 否 string 原支付密码，修改密码时必填

响应参数说明：

参数名 类型 说明
code int 状态码，0表示成功
message string 提示信息
data string 操作结果提示信息

    "message": "success"
}

{

    "data": "支付密码修改成功",

错误码 说明

  2. 首次设置支付密码时，不需要传入原支付密码参数
  3. 修改支付密码时，必须提供正确的原支付密码进行验证
  4. 为安全起见，建议前端对支付密码进行加密后再传输

验证支付密码

接口描述：在进行涉及钱包资金变动的操作（如支付、提现等）时，验证用户输入的支付密码是否正确。

请求路径：/api/wallet/payment-password/verify

请求方法：POST

请求头：

Content-Type: application/json
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

请求参数说明：

参数名 必选 类型 说明
userId 是 long 用户ID
paymentPassword 是 string 待验证的支付密码

请求示例：

{
    "userId": 123,
    "paymentPassword": "123456"
}

响应参数说明：

参数名 类型 说明
code int 状态码，0表示成功
message string 提示信息
data boolean 验证结果，true表示验证通过，false表示验证失败

{
    "code": 0,
    "data": true,
    "message": "success"
}

响应示例(密码错误)：
